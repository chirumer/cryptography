## Summary of Key Concepts

### Local Differential Privacy (LDP)
- Provides privacy guarantees without a trusted data curator
- Each user perturbs their own data before sending to the aggregator
- Privacy parameter ε controls the privacy-utility trade-off

### LDP Protocols
- **k-RR**: k-ary Randomized Response - simple protocol for small domains
- **OUE**: Optimized Unary Encoding - better for large domains, higher communication cost
- **OLH**: Optimized Local Hashing - better for large domains, lower communication cost
- **RAPPOR**: Uses permanent + instantaneous randomization for longitudinal privacy

### Data Poisoning Attacks
- **MGA**: Maximal Gain Attack - maximizes frequency boost for target item
- **RIA**: Random Input Attack - uses random inputs
- **RPA**: Random Perturbation Attack - uses random perturbations

### Key Parameters
- **n**: Number of genuine users
- **m**: Number of malicious/fake users  
- **p**: True positive probability (user's value is supported)
- **q**: False positive probability (other values are supported)
- **ε**: Privacy parameter
- **d**: Domain size
